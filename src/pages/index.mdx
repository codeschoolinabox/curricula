# Spir@Learn

A free, open-source curriculum for learning to program — built on the principle
that you should measure your progress by how well you **understand** code, not
how much you **produce**.

Spiralearn grows from eight years of curriculum design experimentation, watching
where learners get stuck, and research translation. The pattern is always the
same: courses push people to build more, faster, with barely any guidance on how
to understand what they're writing. Spiralearn flips this. You read and trace
code before you write it. You build a vocabulary for describing what programs do
before you're asked to make them do new things.

Most courses also teach code in isolation — stripped from the collaborative and
communication networks it actually lives in. Spiralearn introduces code as an
act of communication from the start, presenting the full picture of who code is
written for and why, while stripping away technical complexity so learners can
focus on process skills and the big picture.

And when AI enters the picture, you're prepared — not just to read AI-generated
code, but to understand and communicate with AI as a genuine collaborator. The
comprehension skills this curriculum builds are exactly the skills that make AI
collaboration effective.

The curriculum follows a spiral structure: you revisit the same skills at
increasing depth and complexity. Each time you return, new connections form
between what you already know and what you're learning. These connections aren't
side effects — they're the core of what it means to learn programming. If a
course has 4 concepts, there are at minimum 10 learning objectives: 4 concepts
plus 6 connections between them. The spiral is what makes this explicit practice
possible.

## What Makes This Different

Most introductory programming courses teach language features in sequence,
present you with challenges to solve, then hope understanding follows.
Spiralearn is structured around four ideas that make it fundamentally different:

**Code has three primary audiences.** Every program simultaneously speaks to
other developers (through naming, comments, and structure), to a computer
(through precise instructions), and to end users (through interfaces and
behavior). The curriculum adds one audience at a time across its chapters, so
learners build a complete picture of what programming actually is — not just
"making the computer do things."

**Connections are concepts.** If a course has 4 concepts, it has at minimum 10
learning objectives: 4 concepts plus 6 connections between them. Most courses
teach concepts in isolation and leave learners to integrate them on their own
during projects. Spiralearn names and practices every connection explicitly.
This is why the spiral structure matters — you can't practice connections until
you have multiple concepts to connect.

**Comprehension before production.** For each new concept, learners follow an
arc: read, trace, describe, modify, then write. Writing code from scratch comes
last, after you can already explain what the code does at multiple levels and
have built a mental library of coding strategies. This isn't training wheels —
it's how we learn most efficiently, and it's direct preparation for a world
where reading and evaluating code (including AI-generated code) is becoming more
important than writing it from scratch.

**Code is content.** Spiralearn treats every line of code as raw learning
material for a variety of interactive exercises — trace tables, syntax marking,
predictive stepping, Parsons problems, program completion, and more. Each
exercise type targets a different skill along the comprehension-to-production
continuum.
[Study Lenses](https://github.com/codeschoolinabox/open-course-study-lenses-committee)
generates this full battery of exercises from any code snippet, whether it's a
curriculum example, a learner's own side-project, or code generated by an LLM.

## The Introductory Curriculum

[**Welcome to Programming**](https://codeschoolinabox.github.io/spiralearn/welcome-to-programming/)
is Spiralearn's introductory sequence: 5 chapters carrying learners from no
experience to an introduction to algorithms, using the minimal possible set of
language features.

Each chapter adds an audience to the learner's awareness:

| Chapter                 | What's New                                                |
| ----------------------- | --------------------------------------------------------- |
| 0. What is Programming? | Orientation and framing                                   |
| 1. Developers ...       | Code as communication between people                      |
| 2. + Computers ...      | Running programs, tracing execution, memory               |
| 3. + Users ...          | Input/output, conditionals, loops, full programs          |
| 4. + Agents ...         | AI as collaborator — theory of mind, collaboration skills |

The curriculum includes
[Study Lenses](https://github.com/codeschoolinabox/open-course-study-lenses-committee)
— interactive exercises generated from code examples. Learners don't just read
about programs, they mark syntax, trace execution step by step, predict program
state, fill in blanks, reassemble shuffled lines, and compare different
implementations of the same behavior.

Content is designed for JavaScript (browser) and Python, and can be studied
alone or in a group — no instructor required.

## How We're Organized

Spiralearn is built by a small team of collaborators, many of whom are MIT
Emerging Talent alumni. The work is organized into four committees, each owning
a different aspect of the project:

**[Storytelling Committee](https://github.com/codeschoolinabox/open-course-storytelling-committee)**
— Who are we, what do we do, and why should you care? Owns the landing page,
introductory video, brand identity, and the framing that helps learners
understand why this curriculum exists and what makes it worth their time. Works
closely with Curriculum to make content engaging and to reach our target
learners.

**[Curriculum Committee](https://github.com/codeschoolinabox/open-course-curriculum-committee)**
— What do we teach and how do we teach it? Owns the curriculum content,
interactive exercises, and instructional videos. Builds directly from teaching
experience, informed by computing education research. This is where the core
product lives.

**[Research Committee](https://github.com/codeschoolinabox/open-course-research-committee)**
— Keeps the curriculum grounded in evidence. Ensures our methods are informed by
recent computing education research, and that we have a data-based understanding
of our own reach and effectiveness. Helps Curriculum stay relevant and
differentiated.

**[Study Lenses Committee](https://github.com/codeschoolinabox/open-course-study-lenses-committee)**
— Builds the exercise platform and developer infrastructure. Study Lenses
generates interactive exercises from raw code examples — the technology that
turns a code snippet into a full battery of comprehension activities. Owns the
tooling that will eventually power rapid content creation and learning
analytics.

All committees share one
[project board](https://github.com/orgs/codeschoolinabox/projects/1) for
tracking work across repositories.

### How the committees connect

Storytelling and Curriculum are on the critical path to launch — there needs to
be something worth visiting (Storytelling) and something worth studying
(Curriculum). Research informs the curriculum but doesn't block it. Study Lenses
will eventually provide the full exercise platform, but the initial curriculum
can integrate prototype exercises as embedded components without waiting for the
complete tooling.

```
Storytelling ──→ the front door (landing page, video, framing)
                    ↓
Curriculum ────→ the product (content, exercises, videos)
                    ↑               ↑
Research ──────→ evidence       Study Lenses ──→ exercise platform
(informs)                       (accelerates)
```
