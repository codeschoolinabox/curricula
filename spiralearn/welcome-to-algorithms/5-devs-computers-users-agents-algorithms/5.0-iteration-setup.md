# 5.0: Iteration Setup

Pure loop mechanics — no algorithm study, no representation sequence.

This subchapter introduces the new loop constructs and explores the relationship
between different iteration approaches. It is **setup for algorithm study**, not
algorithm study itself — the full representation sequence does not apply here
because these programs don't yet reach the algorithmic complexity that warrants
multiple representations. Learners build fluency with iteration patterns that
will be the building blocks of every algorithm in subsequent subchapters.

- `for...of` (JS) / `for...in` (Python) — sequential character iteration
- `while` with manual counter — the "deconstructed for loop"
- `for` with incrementer (JS) / `for...in range` (Python) — native counter
  support
- Refactoring between iteration approaches (sugar vs. capability)
- Understanding when you need index access and when you don't

## Skills and Objectives

- 🥚 Writing and tracing `for...of` (JS) / `for...in` (Python) loops that
  iterate through each character in a string
- 🥚 Writing and tracing `for` with incrementer (JS) / `for...in range` (Python)
  loops with index access
- 🥚 Writing and tracing `while` loops with a manual counter — understanding
  this as the "deconstructed for loop"
- 🐣 Refactoring between iteration approaches: recognizing which is syntactic
  sugar and which adds capability (index access)
- 🐣 Choosing the right loop type for a given task based on whether index access
  is needed
