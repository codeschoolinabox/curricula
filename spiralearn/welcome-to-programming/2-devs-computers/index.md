# Chapter 2: Developers, and the Computer

> The second audience enters: the computer. Source code is now both a text for developers to read and a set of instructions for the computer to execute.

## Overview

This chapter introduces the distinction between **static source code** (a text file) and **dynamic program execution** (the computer carrying out instructions). This is one of the most fundamental conceptual shifts in learning to program, and it's where the debugger becomes a learner's closest study partner.

By the end of this chapter, learners will have encountered all the data types and operators they'll need for the entire curriculum. The progression is deliberate: first see output (logging), then store values (variables), then make claims about values (asserting), then work with the full type system. Each step adds one new dimension while spiraling back through everything that came before.

### Studying with LLMs

As exercise types expand in this chapter (marking syntax, tracing, predictive stepping, fill-in-the-blanks, assertion exercises), so do the available LLM Study Strategies. Learners can now use LLMs to:

- Generate traces to compare against their own predictions
- Quiz them on program state and expression evaluation
- Generate programs with blanks for additional practice
- Explain syntax they find confusing, using the curriculum's vocabulary

See the "LLM Study Strategy" sections in [`../-1-getting-started/exercise-types.md`](../-1-getting-started/exercise-types.md) for starter prompts. Learners can also attach the LLM context documents from [`../-1-getting-started/studying-with-llms/`](../-1-getting-started/studying-with-llms/) to their LLM sessions to keep the AI focused on the curriculum's language subset and teaching approach.
