# 2.3: Asserting

Assertions replace logging as the primary way to test understanding of program memory. Instead of printing values and visually checking them, learners write code that _claims_ what should be in memory — and the program tells them if they're wrong.

This shifts from using the console as a "crutch" to directly reasoning about program state through code.

## Language Features

- Comparison operators (`===`, `!==` in JS; `==`, `!=` in Python)
- `console.assert` (JS) / `assert` (Python)

## Skills and Objectives

- 🥚 Predicting execution without a debugger, by writing code that passes assertions
- 🥚 Writing assertions that describe what's in memory at specific points
- 🐣 Writing lines of code to pass assertions sprinkled through a short script (a step toward writing programs, but still focused on comprehension — you must understand the existing code to write lines that satisfy the assertions)

## Exercise Types

- Tracing (predicting assertion pass/fail)
- Completing programs (writing code to satisfy assertions)
- Predictive stepping (programs with assertions)
- Comparing programs (different assertion strategies)
