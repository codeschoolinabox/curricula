# Chapter 1: Developers

> The first audience: other developers. Source code is a shared text, and
> communication starts with comments.

## Overview

This is the gentlest possible entry into writing code. The only language feature
is comments — text that the computer ignores but developers read. This may seem
trivially simple, but it establishes a principle that runs through the entire
curriculum: **code is written first for humans, then for computers**.

By starting with comments alone, learners practice the developer-to-developer
communication channel in isolation. They write descriptions of what programs
should do and why, without any distraction from syntax, execution, or debugging.
They also begin building IDE fluency and learn to share their work.

## Language Features

- **Comments** (single-line and block comments in JS and Python)

## Skills and Objectives

- 🥚 Writing comments that describe what a program should do and why it should
  do it
- 🥚 IDE skills: opening files, editing files, saving files
- 🥚 Code conventions and style basics
  - Formatting (minimal for comments-only programs, but establishes the
    principle: comments start at the beginning of a line, consistent
    indentation, etc.)
- 🐣 Sharing code with others (GitHub Gists proposed — introduces the concept of
  code as a shareable text)

## Exercise Types at This Stage

The exercise types available are limited since there's no executable code yet:

- **Marking syntax** — identifying comment syntax (single-line vs. block)
- **Reading and writing** — reading comments aloud, writing comments from
  dictation
- **Formatting** — applying consistent style to comment-only files

## Studying with LLMs

This is where learners first encounter the LLM study material from
[`../-1-getting-started/studying-with-llms/`](../-1-getting-started/studying-with-llms/).
Since there is no executable code yet, LLM interaction takes its gentlest
possible form: discussing comments, program descriptions, and comprehension
questions in natural language. Learners are introduced to the general
collaboration strategies, the template study prompt, and responsible use
awareness (hallucinations, energy consumption, copyright concerns).

The key principle established here: AI is a study partner, not a substitute for
your own understanding. You bring the direction; the AI brings additional
perspective.
