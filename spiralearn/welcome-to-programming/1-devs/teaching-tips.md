# Teaching Tips: Chapter 1

## Design Notes

- **IDE question**: If the curriculum requires IDEs, this is where the
  requirement kicks in. If not, learners could use any text editor. The
  tradeoff: IDEs provide helpful features (syntax highlighting for comments,
  formatting on save) but add complexity.
- **Sharing mechanism**: GitHub Gists are proposed for sharing, which will be
  supported by a Study Lenses dependency library for micromaterial development.
- **Seeming simplicity**: This chapter may feel too easy to some learners.
  That's intentional — the goal is to build confidence with the tools and
  establish the "code is communication" mindset before adding technical
  complexity. The real learning here is conceptual, not technical.
