# 3.1: User Input and Output

Users can now interact with the program. This is the minimal viable
demonstration of the full rhetorical triangle — developers write code, the
computer executes it, and users experience the result.

## Language Features

- **JS (browser)**: `prompt`, `alert`, `confirm` (native browser APIs)
- **Python**: `input`

## Skills and Objectives

- 🥚 Top-level doc comments describing the program's name, purpose, and behavior
- 🥚 "Why" not "what" comments — now that programs have enough substance to
  merit them, comments should explain why a line contributes to the user's
  experience (the "why" is usually about the user — the ultimate purpose of the
  program per the rhetorical model)
- 🥚 Writing simple programs that check something about the input or do
  madlibs-type string operations

## Exercise Types

- All prior exercise types (marking syntax, tracing, predictive stepping,
  completing programs)
- Describing programs: zooming out (user's journey, inputs/outputs, paths)
- **Generative AI in the mix** — now that the full rhetorical triangle is active
  (developers, computer, users), this is the natural place to explore how AI
  fits into the picture. Using the thought experiments from
  [`../4-devs-computers-users-agents/gen-ai-in-the-mix.md`](../4-devs-computers-users-agents/gen-ai-in-the-mix.md),
  learners study and discuss: How does AI change each person's role? Has the
  relationship between developers and users changed? Who is most impacted? Does
  AI fundamentally change what it means to develop software? What does AI's
  placement _outside_ the main circle suggest?
