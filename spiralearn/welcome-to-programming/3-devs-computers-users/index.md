# Chapter 3: You, Other Developers, the Computer, and Users

> The third audience completes the rhetorical triangle: users. Programs now have a purpose beyond the developer's console — they take input, process it, and produce output for real people.

## Overview

This is the largest and most skill-dense chapter. With users in the picture, programs gain a "why" that goes beyond technical exercise. Comments shift from "what does this do?" to "why does this exist for the user?". Test cases become descriptions of user-facing behavior. And the full development cycle — from understanding a spec to writing, debugging, and reviewing code — unfolds.

The chapter follows a carefully sequenced progression:

1. **User I/O** (3.1) — the user enters the picture
2. **Conditionals** (3.2) — programs can respond differently to different inputs
3. **Input validation** (3.3) — programs ensure input meets requirements before proceeding
4. **PBSI** (3.4) — a framework for understanding programs at four levels of abstraction (a major milestone)
5. **Developing programs** (3.5) — the full arc from modification to writing from scratch
6. **Plaintext programs** (3.6) — a fun interlude stripping away all IDE assistance

By the end of this chapter, learners can read, trace, describe, modify, review, and write complete interactive programs. They are ready for algorithms.
