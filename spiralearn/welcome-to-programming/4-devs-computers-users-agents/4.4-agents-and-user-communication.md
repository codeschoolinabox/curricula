# 4.4: Agents and User Communication

Revisits Chapter 3 skills (full programs, PBSI, testing, debugging, code review) through the collaboration lens. This is the convergence point — all previous rings come together. This is the threshold moment.

## Skills and Objectives

- 🥚 Read LLM-generated code and identify what it does (using Ch 1–3 skills)
- 🐣 Apply full PBSI evaluation to LLM-generated programs:
  - **Purpose:** Does the generated program solve the right problem?
  - **Behavior:** Does it handle user interaction correctly?
  - **Strategy:** Is the approach reasonable?
  - **Implementation:** Is the code correct line-by-line?
- 🐣 Code review LLM-generated code using the established code review framework from Ch 3.5
- 🐣 Design test cases for LLM-generated programs
- 🐣 Describe gaps between your intent and LLM output using PBSI vocabulary
- 🐥 Debug LLM-generated code: detect bugs, identify root causes, fix them
- 🐥 **Full documentation generation and review:** Interact with LLMs around the complete documentation pattern from Ch 3 — title, prose description, user stories, test cases, structured block comments. Can the LLM generate correct documentation from a spec? Can it generate code from documentation? Can it generate test cases from user stories? Evaluate all outputs using the skills practiced throughout this chapter. This is where developer communication (4.2), computer-facing correctness (4.3), and user-facing completeness (4.4) converge.
- 🐥 **Iterate:** Full collaboration loop — prompt → evaluate → refine → re-prompt
- 🐔 **SOLO check-in (threshold moment):** By this subchapter, learners have practiced all collaboration skills across all code complexity levels. The decision framework should feel intuitive, not like new content — it's a name for what they've been doing. Reflect: given a new programming task, which collaboration approach would you choose, and why?

## Exercise Types

- Have LLM generate complete interactive programs → evaluate using PBSI at all four levels
- Code review exercises: review LLM-generated code using the structured review template from Ch 3.5 (behavior, goals, comments, linting, variables, logs, logic)
- Test case design: given LLM-generated code, write test cases that cover all conditional paths. Does the generated code actually pass your tests?
- Debugging exercises: LLM-generated programs with subtle bugs. Use Ch 3.2 structured debugging approach (what should it do? → what does it do? → how does it work? → which lines surprise you?)
- Documentation round-trip: write spec → LLM generates documentation → evaluate → LLM generates code from documentation → evaluate → compare to original intent
- Refactoring exercises: have LLM refactor a program (change strategy, preserve behavior). Evaluate using PBSI — did the behavior actually stay the same?
- Full collaboration loop: start with a program idea → articulate → prompt → evaluate → iterate → test → debug → deliver. The complete cycle.
