# 4.5: Looking Back, Looking Forward

A brief pause to name what learners have been practicing. This is NOT a first
introduction of anything — all collaboration skills, SOLO awareness, and
delegation judgment have been woven through subchapters 4.1–4.4.

## Skills and Objectives

- 🐣 Given scenarios, identify which collaboration approach you'd use and why
  (diagnostic, not prescriptive — learners should already have intuitions from
  practice)
- 🐣 Articulate programming concepts precisely enough for an LLM to act on them
- 🐥 Choose appropriate collaboration approach based on learning goals (SOLO
  decision)
- 🐥 Apply perspective-taking to evaluate LLM output in unfamiliar domains
- 🐔 Delegate effectively — identify which parts of a task benefit from LLM
  collaboration
- 🐔 Compare LLM "theory of mind" to human theory of mind — what transfers, what
  doesn't

## Content

- **Reflection quiz/exercise:** Given scenarios at different complexity levels,
  identify which collaboration approach you'd use and why. This should feel like
  naming something you already know, not learning something new.
- **Naming what you've learned:** Formalize the vocabulary — Perspective-Take,
  Calibrate, Articulate, Iterate, Delegate — as labels for skills already
  practiced.
- **Forward connection:** How will these collaboration skills apply when
  algorithms get more complex in Chapters 5–6? When the Strategy level of PBSI
  becomes the primary focus, what role can agents play?
- **Explicit scope statement:** "We've learned to collaborate with agents as
  thinking tools in a human-driven development process. Agents can do far more
  (autonomous coding, agentic workflows) — that's beyond this curriculum's
  scope."
