# 5.3: Composed Scans (Multi-Pass)

**Smallest problem:** A string so short that the first pass's answer is obvious — this reveals what information the second pass actually needs. For "normalize casing" with one character: the count is trivially 1 upper or 1 lower, and the second pass has only one character to convert. The smallest case exposes the INTERFACE between passes: what does pass 2 need from pass 1?

**Growth pattern:** hierarchical decomposition — "solve problem A first, then use A's answer to solve problem B." The overall problem decomposes into sub-problems that must be solved in sequence, where each sub-problem is a 5.1/5.2-level scan.

**State:** the output of one pass becomes input/context for the next — follows from the hierarchy: earlier sub-problem results are the "given" for later sub-problems.

**Scope of attention:** information from a complete prior scan informs the current pass — follows from the decomposition: each pass sees the whole string, but KNOWS something from the previous pass.

**Iteration follows as:** multiple sequential loops — each pass is a 5.1/5.2-level operation, but the composition produces answers that no single pass could.

The bridge between local dependency (5.2) and global relationships (5.4). The algorithm can't answer its question in one pass, but it doesn't need to examine every pair of positions either. Instead, it decomposes the problem: the first pass extracts a summary, the second pass uses that summary.

## Algorithms

- Which vowel is most common? (5 passes counting each vowel, then compare)
- Normalize casing: convert the string to whichever case (upper/lower) is more common (first pass: count upper vs. lower; second pass: convert all to majority)
- Does string A contain all the characters in string B? (for each character in B, scan A to check — structured as sequential passes, not nested logic)
- Compare two strings: which has more vowels? (two independent scans, then compare results)

  5.3 is deliberately lighter on algorithm count. This is a pause point for deepening, interlacing, and cross-subchapter comparison rather than algorithm quantity.

## Decision Complexity Gradient (internal sequencing)

1. **Per-pass decisions are 5.1/5.2 level** — each pass is a familiar scan. The new decision skill is _between_ passes: what to do with the result of sub-problem A when solving sub-problem B.
2. **Two-pass** (count then act): normalize casing — first pass counts, second pass transforms based on count result
3. **Multi-pass with comparison** (count multiple things then compare): most common vowel — five counts, then a comparison decision across results
4. **Cross-string** (scan one string to answer a question about another): "does A contain all chars in B?" — decision involves relating information from two different sources

## Skills and Objectives

- 🥚 Identifying the hierarchical growth pattern: solve sub-problem A first, then use A's result to solve sub-problem B
- 🥚 Recognizing that each individual pass in a composed scan is a 5.1 or 5.2-level operation — the novelty is the composition, not the individual scans
- 🐣 Identifying the interface between passes: what information does pass 2 need from pass 1? How is it communicated?
- 🐣 Explaining the 5.2→5.3 comparison: why "most common vowel" needs multiple passes while "longest run" doesn't — hierarchical vs. sequential dependency
- 🐣 Explaining the 5.3→5.4 comparison: how "does A contain all chars in B?" can be decomposed as sequential scans (5.3) or nested loops (5.4) — and what each approach looks like
- 🐥 Designing a multi-pass decomposition for a novel problem: identifying which sub-problem to solve first and what information it must extract for the next pass

**Why these together:** They all have the same subproblem structure: the overall problem decomposes into sub-problems that are solved sequentially, where later sub-problems depend on earlier results. The key new concept is that state between passes IS the result of a complete prior sub-problem — not just running state from seeing the previous character.

**Key comparison moment (5.2→5.3):** "longest run" needs one pass because the subproblem at each position only depends on the previous position. "Most common vowel" needs multiple passes because the subproblem ("which vowel is most common?") decomposes hierarchically: first count each, then compare counts. Same single-pass skill from 5.1–5.2, composed into multi-pass strategies.

**Key comparison moment (5.3→5.4):** Some hierarchical decompositions can also be solved with nested loops (and vice versa). "Does string A contain all characters in string B?" can be decomposed as sequential scans (one per character in B) or as a nested loop. Comparing the two approaches reveals the relationship between hierarchical and nested subproblem structures.
