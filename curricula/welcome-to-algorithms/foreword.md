# Foreword

## For Learners

This curriculum picks up where [Welcome to Programming](/welcome-to-programming/) left off. You can read code, trace execution, describe program behavior at multiple levels, and collaborate with AI tools using a framework grounded in perspective-taking.

Now you'll apply all of that to algorithms — the strategies programs use to solve problems. You'll learn to understand algorithms in multiple ways (prose, flowcharts, pseudocode, and more), wrap them in named and tested functions, and eventually measure how much work they do.

**What to expect:**

- You'll study string algorithms organized by how they decompose problems — from simple independent-step patterns to nested and converging techniques
- You'll learn to write JSDoc documentation, define functions, and write unit tests
- You'll analyze algorithm efficiency through step-counting and Big O notation
- Your LLM collaboration skills from WtP become tools for critically evaluating algorithm representations

**You might notice** small linked emoji (🔬, 📐, 🧪) scattered through the content. These are evidence tags — they tell educators and researchers how confident we are in each teaching choice. You can safely ignore them.

## For Educators

This curriculum continues the evidence-informed approach from Welcome to Programming. The same evidence tag system applies:

- 🔬 **Established** — Direct research backing
- 📐 **Translated** — Established theory applied to a new context
- 🧪 **Extension** — Derived from experience or untested prediction

Each chapter has a `research-framing.md` explaining the evidence base and design rationale. See the [Research Framing](./research-framing.md) for the curriculum-level overview.

### Design Principle

When we face a choice between what's best for learning and what's best for research observability, learning wins. Always.
