# 2.2: Program State

Now programs can store values and combine them. Variables introduce the concept of **program memory** — data that exists during execution and can change over time.

## Language Features

- Variables (`let` in JS, assignment in Python)
- String concatenation
- `undefined` (JS only — unavoidable, not pedagogically chosen)

## Skills and Objectives

- 🥚 Declaring, assigning, reading, and reassigning variables
- 🥚 Predictive stepping with variables in memory (watching values change in the debugger's scopes panel)
- 🥚 Logging different variable values to the console
- 🥚 Variable names:
  - The computer only cares about the use pattern (where a variable is declared, read, assigned)
  - The actual name is a label to help developers — it's a communication choice
  - Naming conventions: snake_case, PascalCase, camelCase, CONSTANT_CASE
- 🥚 **Trace tables** — the foundational concrete representation of program execution. Multiple formats: "steps" tables (every read/write), "values" tables (only assignments). See the trace section in [`../-1-getting-started/exercise-types.md`](../-1-getting-started/exercise-types.md).

## Exercise Types

- Marking syntax (variables, assignment, concatenation)
- Tracing (trace tables in multiple formats)
- Predictive stepping (programs with variables)
- Completing programs
- Comparing programs (same output, different variable usage)
