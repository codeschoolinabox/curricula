# 2.5: Be the Computer

A fun interlude. These two browser games put you in the shoes of the computer — you read instructions and carry them out, step by step. All that matters is what line you're on _right now_ and which instruction you're reading _right now_. You don't need to understand the whole program at once. Just this step, then the next.

That's the same mental process as predictive stepping, but here the game mechanics give you feedback instead of a debugger.

- 🥚 [compute-it](http://compute-it.toxicode.fr/) — read code, trace through it, determine the correct result. The game doesn't explain the rules — you figure them out by reading the code. That "eureka" moment _is_ the learning.
- 🐣 [little-dot](http://little-dot.toxicode.fr/) — select cards with code on them, interpret the instructions, and navigate a dot through a game world. Trial-and-error discovery, smooth progression, no code to type.

## Language Features

- a pseud-introduction to conditionals and loops.

## Skills and Objectives

- 🥚 Carrying out code instructions as the computer would — one step at a time, one instruction at a time
- 🥚 Experiencing that program execution is sequential and local: the only things that matter are the current instruction and the current state
- 🐣 Figuring out what code does through trial-and-error — both games deliberately don't explain their rules. Reading the code _is_ how you learn the rules.
- 🐣 Building intuition for control flow by physically navigating through a program (little-dot) — a foreshadow of the conditionals and loops coming in Chapter 3
