## PBSI Framework

The Purpose/Behavior/Strategy/Implementation framework provides a 4-level vocabulary for understanding programs. It bridges informal program descriptions to formal algorithmic thinking.

_Citations and evidence to be documented here. Connections to Schulte's Block Model (2008) and other multi-level program comprehension frameworks._

## Program Development Progression

The progression from modification → refactoring → code review → writing from spec follows a comprehension-before-production gradient. Each step requires understanding the previous level before attempting the next.

_Evidence base to be documented here._

## Variable Roles

The variable roles taxonomy (fixed value, stepper, flag, gatherer, holder, temporary) draws from established computing education research.
